# 项目说明

## 用户偏好

- 用户叫红豆。
- 红豆很懒，但是喜欢新技术；协作时应尽量主动、简洁，能直接完成就少让用户手动操作。

## 项目概览

- 这是一个基于 Vite 和 Vue 3 的餐厅首页小项目。
- 页面主题是“栖味小馆”，面向餐厅展示和晚餐预约场景。
- 项目是前端静态单页应用，没有后端服务、数据库、路由或状态管理库。

## 技术栈

- 构建工具：Vite 7。
- 前端框架：Vue 3，使用 `<script setup>` 写法。
- 样式：原生 CSS，集中在 `src/styles.css`。
- 入口文件：`src/main.js` 挂载 Vue 应用到 `#app`。

## 目录结构

- `index.html`：HTML 入口，设置中文语言、移动端视口、页面标题和描述。
- `src/main.js`：创建并挂载 Vue 应用，同时引入全局样式。
- `src/App.vue`：主页面组件，包含导航、首屏、菜单、故事、到店信息和预约表单。
- `src/styles.css`：全局样式和响应式布局。
- `src/assets/restaurant-hero.png`：餐厅首屏背景图。
- `package.json`：项目脚本和依赖声明。
- `vite.config.js`：Vite 配置，启用 Vue 插件。

## 常用命令

- 安装依赖：`npm install`
- 本地开发：`npm run dev`
- 构建生产版本：`npm run build`
- 预览构建结果：`npm run preview`

## 页面功能

- 顶部固定导航，包含品牌、锚点导航和预约入口。
- 首屏使用餐厅背景图和深色遮罩，突出品牌名与主要操作按钮。
- 推荐菜单使用本地数据渲染四道菜品，菜品图片目前由 CSS 渐变模拟。
- 品牌故事区介绍餐厅风味，并展示两人晚餐组合。
- 到店信息区展示地址、营业时间和电话。
- 预约表单支持姓名、日期、时间和人数输入；提交后在页面内显示收到预约请求的状态。
- 日期默认使用当天，并限制不能选择早于当天的日期。

## 开发注意事项

- 目前所有业务数据都写在 `src/App.vue` 的数组和响应式对象中，适合小项目快速修改。
- 页面文案以中文为主，改动时注意保持 `UTF-8` 编码，避免中文乱码。
- 样式已经包含桌面、平板和手机断点；新增内容时要检查移动端布局。
- 现阶段没有自动化测试配置，改动后至少运行 `npm run build` 验证构建是否通过。
- 如果要继续扩展，可以优先考虑把菜单、到店信息、预约时间段拆成独立数据文件或组件。
